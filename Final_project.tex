% Slide 2: Project Description
\begin{frame}[t]{Compromised vehicle threat propagation simulator}
    \vspace{0.5cm}
    \small
    Design and implement an application (using C language) that simulates how a malware propagates across a Vehicle-to-Vehicle (V2V) communication network. Vehicles are represented as nodes, and communication links as edges with weights capturing propagation characteristics. Edge weights may represent:

    \vspace{0.3cm}
    \begin{itemize}
        \item Propagation time (delay)
        \item Infection probability
        \item Trust or vulnerability score
        \item Any other justified metric
    \end{itemize}

    \vspace{0.3cm}
    Your program must analyse the spread of compromise, identify critical vehicles, and compute optimal propagation routes. Implement at least \textbf{four} of the following graph-based functionalities:

    \vspace{0.2cm}
    \begin{enumerate}
        \item Infection spread wave from an initially compromised vehicle
        \item Fastest or most-likely propagation path to a target
        \item All-pairs propagation analysis
        \item Detection of propagation clusters or communities
        \item Backbone of propagation
        \item Detection of problematic cycles
    \end{enumerate}
\end{frame}
